AI Microservice — Document Analysis API

Developer Integration Guide
v2.0 — PDF Support
Base URL: https://devvetsapimanagement.azure-api.net/AiMicroservice

Overview

The Document Analysis API allows you to submit images or PDF documents for AI-powered analysis. It supports both single-file and batch analysis via HTTP multipart/form-data POST requests. PDF files are automatically detected and converted to page images for analysis.

All endpoints require Bearer token authorization in the request header.

ModelSpeed Parameter

Controls which underlying AI model is used. Case-insensitive.

ValueSpeedUse Case
"fastest" (default) ~1-2s Routine screening, quick triage, high-volume workflows
"smarter" 3-4s+ Deep investigation, complex or clinically critical cases. Significantly higher quality.

PdfStrategy Parameter

When a submitted file is a PDF, the API converts it to page images and analyzes them. The PdfStrategy parameter controls how many pages are analyzed. Case-insensitive.

ValueBehavior
"AllPages" (default) Every page is analyzed individually, then a final AI aggregation call produces one overall classification. If any page is classified as medical, the whole document is marked medical.
"FirstPageOnly" Only the first page is analyzed. Faster and cheaper — suitable when the first page is sufficient to determine document type.
💡 Start with "AllPages" for accurate classification. Switch to "FirstPageOnly" only when speed is critical and you're confident the first page is representative.
ℹ️ For non-PDF files (images), the PdfStrategy parameter is ignored.

Endpoints

POST /api/DocumentAnalysis/Analyze

Analyzes a single file (image or PDF). Returns a DocumentAnalysisResponse.

Request Parameters

ParameterTypeRequiredDescription
FileFile (binary)YesThe image or PDF document to analyze (multipart)
AnalyzeTypestringYesAnalysis type, e.g. "MedicalImage"
ModelSpeedstringNoAI model tier. Default: "fastest"

C# Example — Image

var options = new RestClientOptions(
    "https://devvetsapimanagement.azure-api.net") { MaxTimeout = -1 };
var client = new RestClient(options);
var request = new RestRequest(
    "/AiMicroservice/api/DocumentAnalysis/Analyze", Method.Post);
request.AddHeader("Authorization", "Bearer <your-token>");
request.AlwaysMultipartFormData = true;
request.AddFile("File", "/path/to/your/image.jpg");
request.AddParameter("AnalyzeType", "MedicalImage");
request.AddParameter("ModelSpeed", "fastest");
RestResponse response = await client.ExecuteAsync(request);

POST /api/DocumentAnalysis/AnalyzeBatch

Analyzes multiple files in a single request. All files are processed in parallel. PDF files within the batch are automatically detected and processed according to the PdfStrategy parameter.

Request Parameters

ParameterTypeRequiredDescription
FilesFile[] (binary)YesArray of files to analyze (multipart). Supports images and PDFs.
AnalyzeTypestringYesAnalysis type applied to all files.
ModelSpeedstringNoAI model tier. Default: "fastest"
PdfStrategystringNo"AllPages" (default) or "FirstPageOnly". Applies to all PDFs in the batch.
StudyIdGuidNoOptional study identifier for WebSocket event targeting.

C# Example — Mixed Batch (Images + PDF)

var options = new RestClientOptions(
    "https://devvetsapimanagement.azure-api.net") { MaxTimeout = -1 };
var client = new RestClient(options);
var request = new RestRequest(
    "/AiMicroservice/api/DocumentAnalysis/AnalyzeBatch", Method.Post);
request.AddHeader("Authorization", "Bearer <your-token>");
request.AlwaysMultipartFormData = true;
request.AddFile("Files", "/path/to/xray.jpg");
request.AddFile("Files", "/path/to/labresults.pdf");
request.AddFile("Files", "/path/to/invoice.png");
request.AddParameter("AnalyzeType", "MedicalImage");
request.AddParameter("ModelSpeed", "smarter");
request.AddParameter("PdfStrategy", "AllPages");
RestResponse response = await client.ExecuteAsync(request);

PDF Processing Details

When a PDF file is submitted (detected by MIME type application/pdf or .pdf extension), the following pipeline is executed:

1
Conversion The PDF is converted to PNG images (one per page) at 300 DPI via the internal PDF-to-image conversion service.
2
Per-Page Analysis Each page image is sent to the AI model for classification (bounded by a concurrency limit of 5 parallel calls).
3
Aggregation (AllPages only) A final AI call receives all per-page results and produces one overall classification for the entire document.
4
Medical Override If any individual page is classified as medical, the final result is guaranteed to be marked as medical — even if the aggregation AI disagrees.

Additional Response Fields for PDFs

FieldDescription
pageCountTotal number of pages in the source PDF.
analyzedPagesHow many pages were sent to the AI (1 for FirstPageOnly, all for AllPages).
pageResultsArray of per-page results (AllPages only). Each entry includes pageNumber, isMedical, classification, confidencePct.

Choosing the Right Parameters

ModelSpeed

  • "fastest" (default) — High-volume, real-time, or initial screening. Fast turnaround, suitable for most standard cases.
  • "smarter" — When accuracy matters most. Triggers deeper reasoning. Expect 10-30+ seconds but substantially higher quality. Ideal for ambiguous or clinically critical documents.

PdfStrategy

  • "AllPages" (default) — Best accuracy. Every page is analyzed and aggregated. Use for multi-page medical documents where important information may appear on any page.
  • "FirstPageOnly" — Fastest. Only the first page is analyzed. Use when the first page is sufficient (cover pages, single-page documents, quick triage).
💡 Start with ModelSpeed="fastest" and PdfStrategy="AllPages". Switch to "smarter" for complex cases and "FirstPageOnly" only when speed is critical.

Error Handling

The API always returns HTTP 200. Check the isSuccess field to determine if the call succeeded.

ScenarioBehavior
No file attachedReturns error: "No file provided"
AnalyzeType missingReturns error: "analyzeType is required"
No files in batchReturns error: "No files provided"
Auth missing / invalidReturns HTTP 401 Unauthorized
PDF conversion failsReturns result with notes: "PDF conversion returned no pages"
Invalid PdfStrategy valueFalls back to "AllPages" (default)
Invalid ModelSpeed valueFalls back to default configured model

Error Response Example

{
  "isSuccess": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "analyzeType is required"
  },
  "data": null
}

WebSocket Real-Time Events (Batch)

During batch processing, the API emits WebSocket events so the frontend can update the UI in real time.

EventDescription
IMAGE_ANALYSIS_STARTBatch started. Frontend shows spinners on all rows.
IMAGE_ANALYSIS_ITEM_COMPLETEOne file completed. Frontend updates that row with the result.
IMAGE_ANALYSIS_ERROROne file failed. Frontend shows error on that row.
IMAGE_ANALYSIS_BATCH_COMPLETEAll files done. Frontend updates summary bar and enables Submit button.
