Exam Instructions

Create a new ASP.NET Core Web API project that manages products. Your API should include two endpoints as specified below.

Product Model

Use the following product structure:

public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Description { get; set; }
    public decimal Price { get; set; }
    public int StockQuantity { get; set; }
}

Question 1: JSON File Storage (70 points)

Task

Create a Web API with two endpoints that store and retrieve products from a JSON file.

1.1 GET Endpoint - Get Product by ID (30 points)

  • Route: GET /api/products/{id}
  • Functionality:
    • Read products from a JSON file named products.json
    • Return the product with the matching ID
    • Return appropriate HTTP status codes:
      • 200 OK if product found
      • 404 Not Found if product doesn't exist
      • 500 Internal Server Error if file reading fails

1.2 POST Endpoint - Insert Product (40 points)

  • Route: POST /api/products
  • Functionality:
    • Accept a product object in the request body
    • Read existing products from products.json
    • Auto-generate the next available ID
    • Validate the product data (all fields required, price > 0, stock >= 0)
    • Add the new product to the list
    • Save the updated list back to products.json
    • Return appropriate HTTP status codes:
      • 201 Created with the created product
      • 400 Bad Request if validation fails
      • 500 Internal Server Error if file operations fail

Bonus Question 1: SQL Database Storage (15 points)

Task

Extend your API to also support SQL Server database storage.

Requirements

Database Setup:

  • Create a SQL Server database named ProductsDB
  • Create a Products table with the following schema:
CREATE TABLE Products (
    Id INT PRIMARY KEY IDENTITY(1,1),
    Name NVARCHAR(100) NOT NULL,
    Description NVARCHAR(500),
    Price DECIMAL(18,2) NOT NULL,
    StockQuantity INT NOT NULL

Implementation (Choose ONE approach):

  • Option A: Entity Framework Core with DbContext

Endpoints:

Bonus Question 2: Object Property Inspector (15 points)

Task

Write a recursive function that prints all properties of an object (including nested objects) to Debug.Log or console.

Requirements

  • Function signature: void PrintObjectProperties(object obj, string indent = "")
  • Print each property in the format: "Property Name": "Property Value"
  • Handle nested objects recursively
  • Handle collections (arrays, lists) appropriately
  • Handle null values gracefully
  • Use proper indentation to show object hierarchy
